Engine Pipeline — Per Message
1
buildPrompt(state, contracts, ledger, consequences)
Assembles system prompt from live state snapshot, AI contracts, compact ledger, active consequences with timers, and module context
2
callAI(messages, systemPrompt)
Provider abstraction: Gemini (primary) → OpenRouter (fallback) → error. Retry + timeout handling. Never depends on one provider.
3
extractMechanics(response)
Parses 65 mechanic keys from AI text. Pattern matching + dispatch table. Each key has registered handler.
4
validateMechanics(changes, state) — LAW 2
9 enforcement gates. Rejects invalid changes. Clamps bounds. Checks ownership. Flags drift.
5
applyMechanics(validChanges, state)
Writes to SolidJS signals → UI reacts → Firebase syncs → checkpoint if significant
The 9 Enforcement Gates
Roll Confirm
No player roll submitted? Reject mechanic.
Combat Turn
Wrong entity's turn? Reject action.
Ownership
AI writing system field? Blocked.
Scene Transit
Location/time change needs player confirm.
PC Action
AI acting for unmentioned PC? Flag.
Drift Detect
Narrated state change without mechanic? Flag.
Concentration
Damage taken → auto-trigger save check.
Timer Enforce
Consequence expired → flag for resolution.
Clamp/Bounds
HP ≤ max, gold ≥ 0, etc.

Three Data Tiers
🔥 Firebase (Synced)
Game state that changes during play:

• HP, conditions, gold
• Quest status, NPC data
• Chat history
• Combat state
• Inventory / wagon
• World data (location, weather, time)
• Contracts
• Treasury
• Consequences

All devices see same state
💿 IndexedDB (Local)
Reference content, never synced:

• Spell compendium
• Class progression data
• Feat database
• Monster data
• Module text / chapters
• Parsed books (PDF/epub)
• Web imports
• Map images
• Glossary (97 terms)

Same on all devices via import
📦 Shared Bundles
Content packs for multiplayer:

• Generated by one player
• Shared via import link
• Others import as they join
• Reusable across campaigns

Firebase only stores:
"has_pack_X": true

Actual content lives in IndexedDB
Sync Model
Normal Flow
State change
  → SolidJS signal
  → UI reacts
  → Firebase.set()
  → Other devices get .on()
Offline Flow
State change
  → SolidJS signal
  → UI reacts
  → localStorage queue
  → reconnect → reconcile
Conflict Resolution
Chat Messages
Clock-independent merge. Messages carry sequence IDs, not timestamps. Proven in v1.
State Fields
Last-write-wins for simple fields. Merge-on-reconnect for arrays (inventory, conditions).
